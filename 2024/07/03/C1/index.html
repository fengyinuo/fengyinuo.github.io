整数存储

在C语言中，整数以二进制补码形式在内存中存储。

负数符号位是1，正数符号位是0

数值范围

最小值 最大值
int -2^31 = −2147483648 2^31 - 1 = 2147483647
unsigned int 0 2^32 - 1 = 4294967295

数值越界

printf不会执行自动类型转换，它会直接根据格式说明符（例如 %d, %u, %f, %s 等）读取并解释传入的参数，而不会根据参数的实际类型进行隐式转换。

正整数 2147483648 转换为 1000 0000 0000 0000 0000 0000 0000 0000（二进制）存储。当用%d解释时，在补码表示法下，由于该二进制数的最高位是 1，它代表的是负数。
所以首先将补码转换回原码 1000 0000 0000 0000 0000 0000 0000 0000（二进制） => 2147483648（十进制），但是由于它是负数，所以我们得到的 -2147483648。

-3 在内存中存储时，使用的是补码表示法，即转换为 1111 1111 1111 1111 1111 1111 1111 1101存储。
当用%u解释时，直接使用二进制数据转换为十进制表示 => 4294967293。
