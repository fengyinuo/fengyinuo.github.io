数组

作者: Suno 时间: July 3, 2024  17:27:00 类别: C

数组在内存中是连续存放的，随着数组下标的增长，地址是由低到高变化的。

遍历数组

数组传参

在C语言中，数组传参时，只能传递数组的首地址（即指向数组第一个元素的指针），而不能直接传递整个数组。
这是因为在C语言中，数组名在大多数情况下都会退化为指向数组首元素的指针。

存在两个意外：

sizeof(数组名)：数组名表示整个数组，计算整个数组的大小，单位是字节的

&数组名，数组名表示整个数组，取出的是整个数组的地址

除此之外，遇到的所有数组名都是首元素地址.

数组地址

arr == &arr[0]，arr和&arr[0]都表示数组首元素的内存地址，两者是等价的，&arr表示整个数组的地址。

数组下标操作符 [] 的定义如下：

a[b] 等价于 *(a + b)

这意味着： arr[2] 等价于 *(arr + 2)。
2[arr] 也等价于 *(2 + arr)，这在数学上等价于 *(arr + 2), 所以，arr[2] 和 2[arr] 是等价的。
在C语言中，arr[2]确实等价于2[arr]。这看起来有些奇怪，但这是因为C语言中的数组下标操作符 [] 是通过指针运算实现的。
